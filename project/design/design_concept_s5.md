## 1. Информационная архитектура / карта сайта

**Принцип:** главная перестаёт быть словарём. Главная = **ответ-вход** + равный доступ к 4 разделам. История выходит из подвала и становится самостоятельной страницей-сиблингом, а не тизером. Слоты «Обучение» и «Сообщество» — заглушки-места в IA, не наполняются.

```
/ (главная)
│  • вордмарк «Культура хоккея» + «при поддержке „Хоккейного Овертайма“»
│  • ОТВЕТ-ВХОД: поле «спроси» (hero, главный элемент экрана)
│  • 4 равноправных раздела (карточки-входы):
│       Словарь · История · Обучение(слот) · Сообщество(слот)
│  • «популярное / с чего начать» (3–4 курируемых входа)
│
├── /dictionary  (Словарь)
│     • поиск внутри словаря
│     • фильтры: Уровень × Профиль × Категория (3 независимые оси)
│     • кластеры терминов (вложенность) + плоский A→Z как альт-вид
│     • карточка термина: связи «см. также» + якорь в историю/гайд + слот визуала
│
├── /history  (История — индекс)
│     • блоки: Рождение · Советский хоккей · Современный
│     • карточки-тизеры глав (статус: опубликована / «скоро»)
│     └── /history/<chapter>.html  (страница главы)
│            • h2-разделы с id-якорями (цель deep-link от ответ-входа)
│            • fact-box, таймлайн, nav-footer «← пред · след →»
│
├── /learn      (Обучение — СЛОТ, страница-заглушка «скоро»)
└── /community  (Сообщество — СЛОТ, страница-заглушка «скоро»)
```

**Глобальная навигация.** Постоянная шапка: вордмарк слева + 4 раздела (Словарь · История · Обучение · Сообщество). Слоты помечены приглушённо с меткой «скоро» — место видно, наполнения нет. На мобильном — вордмарк + компактная таб-строка (4 пункта) или «гамбургер». Ссылка-партнёр на канал — в шапке (мелко) и в футере.

**Почему так, а не «словарь на главной».** Чтобы История стала *равной*, она должна быть сиблингом главной, а не секцией внутри словаря. Сейчас история — `<section class="history-teaser">` у подвала `index.html`; это и есть «подвал», который чиним. Поэтому словарь переезжает на свою страницу, а главная становится лёгким хабом-маршрутизатором.

**Поэтапность (важно — автор один, ~2 ч/нед).**
- **Шаг 1 (минимальный, самоценный):** текущий `index.html` → переименовать в `dictionary.html`; новый лёгкий `index.html` = ответ-вход + 4 карточки-входа. История уже имеет страницы — просто поднять в навигацию. Слоты = две статичные заглушки. → платформа «ощущается» сразу, ничего не сломано.
- **Шаг 2:** две оси фильтров и кластеры в словаре (требует полей в данных — см. §8).
- **Шаг 3:** связи/якоря в карточках + слоты визуала.
Каждый шаг работает сам по себе и переживает паузу.

## 2. Логика ключевых экранов (вайрфрейм-уровень)

### 2.1 Главная — ответ-вход «спроси»

Гибридный движок (решение PM): сверху — курируемые быстрые ответы на топ-вопросы (ручной реестр Вопрос→якорь), ниже — Fuse.js-поиск по терминам + keywords истории как fallback. Без живого LLM — это маршрутизатор по структуре.

```
┌─────────────────────────────────────────────┐
│  КУЛЬТУРА ХОККЕЯ        Словарь История …    │  ← шапка, вордмарк + nav
│  при поддержке «Хоккейного Овертайма» ↗      │
├─────────────────────────────────────────────┤
│                                             │
│   Спроси про хоккей                         │  ← H1 / подзаголовок-обещание
│   ┌─────────────────────────────────────┐   │
│   │ 🔍  что такое форчек?            ↵  │   │  ← ОТВЕТ-ВХОД (hero)
│   └─────────────────────────────────────┘   │
│   Часто спрашивают:                         │  ← подсказки (idle)
│   [что такое xG] [откуда хоккей] [слот]     │  ← курируемые чипсы-вопросы
│                                             │
│   ── Разделы ──                             │
│   ┌──────────┐ ┌──────────┐                 │
│   │ СЛОВАРЬ  │ │ ИСТОРИЯ   │                 │  ← 4 равных входа
│   │ 49 терм. │ │ 9 глав    │                 │
│   └──────────┘ └──────────┘                 │
│   ┌──────────┐ ┌──────────┐                 │
│   │ ОБУЧЕНИЕ │ │СООБЩЕСТВО │                 │  ← слоты, метка «скоро»
│   │  скоро   │ │  скоро    │                 │     (приглушены)
│   └──────────┘ └──────────┘                 │
└─────────────────────────────────────────────┘
```

**Состояния ответ-входа:**

| Состояние | Что показываем |
|---|---|
| **idle / подсказки** | placeholder «термин или тема…» + чипсы курируемых вопросов («что такое xG», «откуда хоккей», «как читать Корси») |
| **быстрый ответ** (курируемый, hybrid-слой) | плашка «⚡ быстрый ответ» сверху выдачи: 1–2 предложения + кнопка «Открыть →» на точный якорь. Перекрывает шум поиска по топ-вопросам |
| **найдено — термин** | карточка: `EN / РУ` + бейдж категории + 1 строка из definition + «В словарь →» (deep-link `dictionary.html#<id>`) |
| **найдено — история** | карточка: «Глава N» + заголовок + summary + «Читать главу →» (deep-link `history/<chapter>.html#<section-id>`) |
| **смешанная выдача** | сначала курируемый быстрый ответ → термины → главы истории. Каждый результат = ссылка-якорь, а не «почитай целиком» |
| **не найдено** | «Пока не нашли точного ответа» + предложить: открыть Словарь / Историю + ссылка «спросить в канале ↗». Без тупика |

Движок отдаёт **саммари + deep-link**, не полный текст — это формула платформы (§6.1 концепта) и причина, почему LLM не нужен.

### 2.2 Словарь — две оси таксономии + кластеры

```
┌─────────────────────────────────────────────┐
│  Словарь                                    │
│  ┌─────────────────────────────────────┐   │
│  │ 🔍  поиск по словарю…               │   │
│  └─────────────────────────────────────┘   │
│                                             │
│  Уровень:  [ База ][ Средний ][ Продвинут.] │  ← ось 1: сегмент, прогрессия
│  Профиль:  (Общий)(Игрок)(Тренер)(Аналитик)(Скаут) │ ← ось 2: чипсы, мультивыбор
│  Категория:(Стат.)(Тактика)(Правила)(Позиции)│  ← ось 3: чипсы (как сейчас)
│  активно: Средний · Тренер · Тактика  [сброс]│  ← сводка активных + сброс
│                                             │
│  ▸ КЛАСТЕР: Броски                  (4)     │  ← группа с вложенностью
│     ├ Кистевой бросок                       │
│     ├ Щелчок                                │
│     └ Подщёлкивание                         │
│  ▸ КЛАСТЕР: Большинство             (3)     │
│     ├ Зонтик                                │
│     └ …                                     │
│  ▸ Вне кластеров (A→Z)              [плоский вид ⇄] │
└─────────────────────────────────────────────┘
```

Оси **независимы и комбинируются**: между осями — И (Средний И Тренер И Тактика), внутри оси Профиль — ИЛИ (Игрок ИЛИ Тренер). Категория остаётся как есть (`stat/tactic/rule/position`). Переключатель «кластеры ⇄ плоский A→Z» — кластеры для обучения, плоский для быстрого поиска (режим энциклопедии vs инструмента, §2 концепта).

## 3. Модель таксономии: Уровень × Профиль × Категория

Три **ортогональные** оси. Термин несёт значение по каждой одновременно.

| Ось | Тип | Значения | Назначение | Сейчас в данных |
|---|---|---|---|---|
| **Категория** | 1 значение | `stat` · `tactic` · `rule` · `position` | что это за сущность | ✅ есть (`category`) |
| **Уровень** | 1 значение, упорядочен | База → Средний → Продвинутый | прогрессия «простое→сложное» | ⚠️ есть `level`, но метки путают (см. ниже) |
| **Профиль** | мультизначение | Общий · Игрок · Тренер · Аналитик · Скаут | кто/зачем пришёл | ❌ новой оси нет |

### Почему текущие теги — «от балды»
Сейчас `level` = `novice / fan / geek`. Это смешивает **уровень** и **роль**: «щелчок = аналитику» — симптом (щелчок нужен игроку, а не аналитику; это вопрос *уровня*, не роли). Разводим в две оси.

### Миграция (рекомендация — без боли в данных)
- **Уровень:** оставить ключи `novice/fan/geek` как внутренние, поменять только **отображаемые метки**: `novice→База`, `fan→Средний`, `geek→Продвинутый`. Поле уже заполнено у всех 49 терминов (20/20/9) — миграция данных не нужна, меняется лейбл-карта в UI. (Если PM захочет — позже переименовать ключи на `base/mid/adv`, но это правка данных, не обязательная.)
- **Профиль:** новое поле-массив `profiles: string[]` у термина. Требует ревизии 49 терминов — это **контент/данные**, задача дев/контент-чата по «рубрикатору присвоения» (§6.5 концепта), не дизайна. Стартовый набор значений: `general, player, coach, analyst, scout`. Скаут уже заложить как значение (слот под рост), наполнять по мере появления терминов.

### Как три оси уживаются в UI
- **Уровень** — сегмент-контрол (ощущение упорядоченной шкалы), одно активное.
- **Профиль** — чипсы с мультивыбором.
- **Категория** — чипсы (как сейчас).
Панель компактная, со строкой «активно: …» и «сброс». На мобильном фильтры сворачиваются в кнопку «Фильтры (2)» с выезжающей панелью.

**Стартовая разметка профилей (пример, не финал — для дев-чата):** Корси/Фенвик/xG → `[analyst]`; Форчек/Бэкчек → `[player, coach, general]`; Слот/Пятак → `[general, player]`; Вбрасывание/Офсайд → `[general]`; Тафгай → `[general]`. Финальную разметку даёт контент-чат по рубрикатору.

## 4. Связи и кластеры терминов

Из плоского списка — в смысловые группы с вложенностью + блок «см. также».

- **Кластеры (вложенность):** родитель-зонтик и дочерние термины. Примеры из ТЗ: `Броски ⊃ {кистевой, щелчок, подщёлкивание}`; `Большинство ⊃ {зонтик, …}`. В UI — сворачиваемый блок-кластер с счётчиком, дочерние с отступом.
- **«См. также»:** горизонтальные связи между терминами (не иерархия): Форчек ↔ Бэкчек ↔ Прессинг. Внизу карточки.
- **Якорь «подробнее»:** связь термина с каноном — конкретное место в главе истории или (на горизонте) в гайде. Это та же «формула платформы»: термин → саммари → якорь в глубину.

Модель связей в данных — §8.

## 5. Паттерн визуала — схема / видео как первоклассный контент

Один компонент-слот, два типа контента. Не украшение — часть ответа (§6.7 концепта).

### Тип A — SVG-схема (своя)
- Стиль уже задан в `docs/diagrams/` (forecheck/slot/powerplay): кремовый фон `#F1EAD9`, заголовки Oswald, подписи PT Mono, чернильные контуры, синяя линия `#4A6E92`. **Держать этот стиль** — это часть ДНК.
- Inline-SVG в карточке/тексте; превью → клик открывает лайтбокс крупно. Подпись + «по мотивам …» при необходимости.
- Делаем сами, near-term реалистично.

### Тип B — видео (чужое, встраиваемое)
- YouTube + **таймкод** как ещё один якорь («смотреть с 2:14»).
- Паттерн «фасад»: показываем превью-кадр + значок ▶, iframe грузится только по клику (перформанс на мобильном, меньше тяжести в сетке).
- Обязательна атрибуция источника. Оригинальный продакшн — на горизонт, сейчас только встраивание.

### Контракт компонента (для дев-чата)
```
visual: {
  type: "scheme" | "video",
  src:   "diagrams/forecheck.svg"  |  "youtube:VIDEO_ID",
  timecode: "2:14"   // только video, опционально → якорь
  caption: "Форчек 2-1-2",
  attribution: "канал … / автор"   // обязателен для video
}
```
Поля `visual` / `visual_type` уже есть в схеме (`scheme/video/gif`) — расширяем смысл, не ломаем.

### Размещение
- В **карточке термина** (свёрнут превью → раскрытие).
- **Inline в тексте главы** (схема по ходу нарратива, видео-вставка).
Один паттерн — оба места.

## 6. Идентичность — 3 направления

Решение PM: базлайн — эволюция ДНК; один смелый вариант на сравнение; финал — по рендерам. Все три просим у Claude Design в одной сессии. Имя в шапке — вордмарк **«Культура хоккея»** + строка «при поддержке „Хоккейного Овертайма“». Узнаваемость с каналом не ломаем. ⚠️ До печати имени на макетах — проверить, не занято ли название (§6.9 концепта).

### A. «Строгий ретро-учебник» — БАЗЛАЙН (эволюция ДНК)
Программка матча, доросшая до учебника. Кремовый фон, Oswald + PT Serif + PT Mono, чернильные линейки — но добавляем структуру учебника: нумерация глав/разделов, маргиналии (заметки на полях), «плашки-таблички» как подписи к схемам, аккуратные оглавления. Максимальная преемственность с каналом. **Низкий риск.**

### B. «Тёплый ламповый» — середина
Мягче и теплее: более тёплый кремовый, больше PT Serif в основном тексте, скруглённые плашки, дружелюбный тон. Ощущение научпопа «Арзамас» — под исторический раздел особенно. Меньше «программки», больше «журнала о хоккее». Средний риск.

### C. «Современный с ретро-акцентами» — СМЕЛЫЙ (на сравнение)
Чистая сетка, больше воздуха, современный гротеск для интерфейсных элементов (фильтры, навигация) + ретро-акценты (эмблема, PT Serif для редакционного текста, чернильные детали дозированно). Платформенный масштаб, ближе к «продукту», чем к «программке». **Самый далёкий от визуала канала — это и есть риск на сравнение.** Решаем по рендеру.
