# Процесс и разделение ролей (чаты)

> Как мы работаем, чтобы не жечь токены: каждый чат держит только свой контекст.
> Версия: 0.1 · 13.06.2026

## Принцип

PM-чат — лёгкий координатор. Он не пишет код, контент и дизайн сам. Он ставит ТЗ,
готовит промпты и маршрутизирует работу в профильные чаты. Тяжёлый контекст
(репозиторий, верстка, исследования) живёт в исполнительских чатах.

## Роли-чаты

| Чат | Что делает | Что НЕ делает |
|---|---|---|
| **PM** (этот) | координация спринтов, приоритеты, ТЗ, промпты, маршрутизация, sprint_*.md | не пишет код / контент / дизайн |
| **Разработчик** (Cowork, репо) | бот, terms.json, history.json, верстка HTML из готовых черновиков, деплой | не исследует, не сочиняет контент истории |
| **Исследователь** | черновики глав истории в markdown по `docs/history/INDEX.md`, факты, цитаты | не верстает, не трогает код |
| **UX/UI дизайнер** | аудит UX, концепция платформы, промпты для Claude Design | не пишет прод-код |
| **Маркетинг/SEO** (по мере надобности) | продвижение, SEO, в т.ч. AI-выдача | — |

## Поток работы спринта

1. **PM** разбирает инбокс → фиксирует scope в `project/planning/sprint_N.md` → режет на задачи.
2. **PM** для каждой задачи вне своей зоны готовит: (а) в какой чат идти/создать, (б) готовый промпт-ТЗ.
3. Исполнительский чат делает работу в своём контексте, отдаёт результат (PR/файл/черновик).
4. **PM** принимает, обновляет статус в sprint-доке и в файле проекта Obsidian.

## Передача между чатами

- Контент истории: **Исследователь** отдаёт markdown-черновик → **Разработчик** верстает HTML.
- Дизайн: **UX-дизайнер** отдаёт концепцию + Claude Design-промпт → **Разработчик** реализует.
- PM не дублирует контекст: в промпт кладём только ссылки на нужные файлы репо, а не их содержимое.
