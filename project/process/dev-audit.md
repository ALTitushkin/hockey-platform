## Гэпы (по приоритету)

### 🟡 P2 — нет глобального error-handler в боте
`bot.py` не регистрирует `application.add_error_handler`. Необработанное исключение в хендлере уходит в лог PTB, пользователь не получает ничего.
**Фикс:** добавить error-handler: залогировать + по возможности ответить «что-то пошло не так, попробуйте ещё раз».

### 🟢 P3 — нет линтера/форматтера
Ни `ruff`/`black`/`flake8`, ни `pre-commit`. Стиль держится вручную (пока хорошо), но линтер ловит мёртвые импорты/переменные и держит единообразие.
**Фикс:** добавить `ruff` (lint+format) + `ruff.toml`; прогон `ruff check` в CI-job `test`; опц. `pre-commit`.

### 🟢 P3 — загрузка данных без try/except
`json.loads(...)` в `search/calendar` падает на битом JSON при старте. Сейчас прикрыто CI (`validate_data` гейтит деплой) → «fail fast» приемлем.
**Фикс (по желанию):** понятный лог при старте, если реестр не читается.

### 🟢 P3 — `build_chapter.py`: конфиг правится в коде
Под каждую главу редактируется блок `CONFIG` в скрипте.
**Фикс (по желанию):** читать конфиг из frontmatter черновика или CLI-аргумента — глава верстается без правки скрипта.

## План рефакторинга

2. P3: `ruff` + конфиг + линт в CI-job `test`.
3. P3: остальное — по мере роста.
