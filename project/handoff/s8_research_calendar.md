## Что изменилось и почему

**Итерация 1 (18.07):** промпт v2 просил закрыть 265 дней одним заходом. За месяц покрытие не сдвинулось (101/366 → 101/366). Диагноз: у объёма такого размера нет точки «сдал → получил обратную связь». Решение — резать на батчи.

**Итерация 2 (19.07):** батч «Август» сдан за один заход (26 новых дней, август 31/31), после чего работа упёрлась в **лимиты**. Вывод владельца: наполнение календаря нельзя держать внутри спринта, оно блокирует всё остальное.

**Текущий режим — фоновый поток.** Календарь наполняется **по одному месяцу за заход**, темпом «сколько позволяют лимиты», без спринтового дедлайна. Ровно как поток глав истории. Спринт от этого потока больше не зависит.

**Что это значит для тебя практически:**
- бери **один месяц** за заход, не три;
- дошёл до лимита — останавливайся и сдавай что есть, это нормальный режим, а не срыв;
- записка к батчу обязательна: она позволяет продолжить с того же места в следующий раз.

## ⚙️ Источники: обход недоступности ru.wikipedia

**Проблема (найдена 19.07):** `ru.wikipedia.org` не открывается ни инструментами исследователя, ни у PM через прямой fetch. А это главный источник по советскому хоккею — то есть по приоритету №1.

**Решение: используй WebSearch как основной путь, а не прямое открытие страниц.** PM проверил на четырёх советских датах августовского батча — поиск возвращает и содержимое ru.wikipedia, и параллельные источники (ХК «Динамо», РИА, ТАСС, championat, km1954.ru, fhr.ru, calend.ru). Для дат рождения и смерти этого достаточно.

**Практика:**
- проверка конкретной даты: `«Валерий Васильев хоккеист дата рождения 3 августа 1949»` — сразу даёт подтверждение или расхождение;
- набор кандидатов на день: `«хоккеисты родившиеся 15 ноября»`, `«советские хоккеисты умерли в ноябре»`;
- расхождение между источниками → `conf: med` + пометка в записке. Пример из августа: Сологубов — часть источников даёт 8 марта, БСЭ 8 августа.

Прочие источники: fhr.ru, khl.ru, championat.com/hockey, sport-express.ru, sports.ru, nhl.com/ru, hockey-reference.com, hhof.com, km1954.ru («Красная машина» — полезен по сборной СССР).
Осторожно с vintagehockeyjerseys.net — там много ошибок в датах, перепроверять по второму источнику.

*Опция на будущее:* подключение Claude in Chrome откроет страницы вида «15 ноября» со списками родившихся — удешевит набор кандидатов. Пока не подключено, WebSearch закрывает задачу.

## Формат сдачи батча

Дописывать прямо в `hockey_calendar/hockey_calendar.json` по ключу `MM-DD`.

```json
"08-15": [
  { "year": 1955, "text": "...", "cat": "Рождение", "reg": "RU", "conf": "high" },
  { "year": 1994, "text": "...", "cat": "Рекорд", "reg": "NHL", "conf": "high",
    "video": "https://www.youtube.com/watch?v=..." }
]
```

`cat`: Рождение / Гибель / Дебют / Рекорд / Матч / Турнир / Переход / Возвращение / Достижение / Организация
`reg`: RU / NHL / INT / WORLD
`conf`: high / med

**Сдавать месяц целиком**, не по одному дню. Упёрся в лимит посреди месяца — сдавай что закрыто и пиши в записке, где остановился.

**Записка к батчу обязательна.** Образец — записка по августу от 19.07: сколько дней закрыто, сколько отечественных, на каких числах пришлось взять зарубежный факт и почему, что осталось сомнительным, где остановился. Именно она позволяет продолжить с того же места в следующий заход.

## DoD батча (любого месяца)

- [ ] Все дни месяца закрыты, пустых нет (или явно указано, где остановился и почему).
- [ ] Каждое событие: точная дата, `cat`/`reg`/`conf` заполнены, источник проверен.
- [ ] Отечественные события стоят первыми в датах, где есть и наше, и зарубежное.
- [ ] Расхождения между источниками помечены `conf: med` и названы в записке.
- [ ] Записка PM: статистика + сомнительные места + точка остановки.
- [ ] `validate_calendar.py` проходит зелёным (после трека A — автоматически в CI).

**Приёмка PM:** формат — скриптом; отечественные даты — выборочно вторым источником через WebSearch. Verify before accept: спорные факты сверяю сам, на слово не принимаю.

**Срока нет.** Фоновый поток: темп задают лимиты, а не календарь спринта. Следующий заход — **ноябрь**.
