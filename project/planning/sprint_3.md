## Цель спринта — одна фраза

Бот живёт на VPS, о нём знают подписчики канала, а каждый ненайденный запрос превращается в задачу на наполнение базы.

---

## Решения, принятые до спринта

- **Хостинг:** существующий британский VPS (1 CPU / 1 GB — боту хватает с запасом). Новый VPS — только когда появится второй сервис.
- **Деплой:** systemd-юнит + GitHub Actions (push в main → ssh → git pull → restart). CI/CD-пайплайн — артефакт для портфолио.
- **Admin-панель, эксперт, визуалки** — не в этом спринте. К эксперту идём со статистикой реальных запросов (спринт 4).

## Заметки по итогам

- VPS: общий с x-ui (VPN) и nginx (опросник), памяти хватает (~230 МБ свободно при работающем боте).
- В .env обязательна строка вида `BOT_TOKEN=...` — голый токен не парсится.
- Грабли инструмента: коммит из Cowork-сессии однажды захватил обрезанный синком файл (4945304) — после коммита проверять, что HEAD-версия компилируется.
- Безопасная смена токена на VPS без редактора: `read -s -p "токен: " T && echo "BOT_TOKEN=$T" > bot/.env && unset T`.
