## Цель спринта — одна фраза

Дать подписчику ежедневный исторический крючок прямо в боте: «что случилось в хоккее в этот день» — на сегодня одной кнопкой и на любую дату текстом.

## Что входит (Sprint 7)

### Трек A · Бот → дев-чат «Разработчик»

**A1. Данные календаря в репозитории**
- Завести `data/calendar.json` (источник — `hockey_calendar/hockey_calendar.json`). Ключ — `MM-DD`, значение — массив событий.
- Поля события: `year`, `text`, `cat`, `reg`, `conf`, **`video`** (опц.: URL на видео гола/момента — см. схему ниже).
- Подключить файл к синку календаря: дев тянет свежий `calendar.json` из артефакта Исследователя при обновлении (ручной коммит на этом спринте, авто-синк — бэклог).

**A2. Логика дат (`search.py` или новый `calendar.py`)**
- `load_calendar()` — грузит и валидирует `data/calendar.json` (как `load_terms()` / `load_history()`).
- `events_for_date("MM-DD") -> list[event]` — все события дня, отсортированы по `year`.
- `parse_date(text) -> "MM-DD" | None` — распознаёт человеческие форматы:
  - `2 января`, `2 янв`, `14 марта`
  - `02.01`, `2.1`, `14.03`, `02-01`
  - игнор регистра и лишних слов («что было **2 января**», «**14 марта** в хоккее»).
- Без года: дата трактуется как «день-месяц» (рубрика по `MM-DD`, год не важен).

**A3. Команды и кнопки**
- Кнопка **«📅 В этот день»** в `/start` (`START_KEYBOARD`) с `callback_data="onthisday:today"` → `CallbackQueryHandler` отвечает событиями на сегодняшнюю дату (UTC/МСК — зафиксировать таймзону, по умолчанию **Europe/Moscow**).
- Команда `/today` (синоним кнопки) и `/day 14-03` (или `/day 14 марта`).
- В `handle_query`: **перед** словарным поиском прогнать `parse_date()`. Если дата распознана — отдать карточку дня и НЕ засорять `missing_queries.log`. Иначе — текущая словарная логика.
- ⚠️ Антиколлизия со словарём: реагируем как на дату только при явном дате-паттерне (число + месяц/разделитель). Одиночные слова («март») словарём не перехватываем как даты.

**A4. Формат карточки (HTML, как `format_card`)**
- Заголовок: `📅 <b>14 марта</b> в истории хоккея`.
- На каждое событие: `<b>{year}</b> — {text}` + строка-метка категории (эмодзи по `cat`).
- Если у события есть `video` → строка-ссылка `🎥 <a href="{video}">Смотреть момент</a>`.
- Несколько событий на дату — все в одном сообщении (компактный список), порядок по году.
- Пусто на дату → дружелюбная заглушка: «На эту дату пока не нашли событие — рубрика ещё наполняется» + кнопка «📅 Случайный день» (опц., если успеваем).
- Футер: ссылка на канал / платформу.

**A5. Тесты и эксплуатация**
- `test_calendar.py` без токена: `parse_date` на наборе форматов (вкл. мусорные), `events_for_date` для известной даты, корректная сортировка, поведение на пустой дате.
- `conf`-политика: на этом спринте показываем все события из `calendar.json` (включая `med`) — данные курируемые; фильтр по `conf` — бэклог, если появятся жалобы.
- CHANGELOG → `v0.7`, README/CLAUDE.md обновить (новые команды, формат `calendar.json`, таймзона).

## Схема события календаря (v1.1 — добавлено поле `video`)

```json
"03-23": [
  {
    "year": 1994,
    "text": "Уэйн Гретцки забил 802-й гол и побил вечный рекорд Горди Хоу по голам в НХЛ.",
    "cat": "Рекорд",
    "reg": "NHL",
    "conf": "high",
    "video": "https://www.youtube.com/watch?of_the_goal"
  }
]
```

- `cat`: Рождение / Гибель / Дебют / Рекорд / Матч / Турнир / Переход / Возвращение / Достижение / Организация
- `reg`: RU / NHL / INT / WORLD
- `conf`: high / med
- `video` (опц.): прямая ссылка на видео момента. Заполняется для голов/шедевров/исторических моментов, где видео существует и проверено. Для «Рождение/Гибель/Организация» обычно отсутствует.

## Зависимости и порядок

1. **A1 → A2 → A3/A4 → A5.** Данные в репо → логика дат → команды/кнопки/карточка → тесты.
2. Параллельно (не блокирует): Исследователь наполняет `hockey_calendar.json` и проставляет `video`. Бот работает на любом срезе данных — чем полнее календарь, тем меньше «заглушек».
3. PM принимает на нескольких датах (вкл. сегодня, дату с несколькими событиями, дату с `video`, пустую дату).
