# Архитектурные решения

## Принцип: данные отдельно от UI

`terms.json` → единственный источник правды. Веб, бот, любой будущий интерфейс — все читают один файл. Поменять дизайн ≠ потерять данные.

## Почему Fuse.js, а не серверный поиск

На 20–500 терминов клиентский поиск быстрее любого round-trip к серверу. При росте базы >1000 терминов — пересмотреть в пользу Typesense или Meilisearch (self-hosted).

## Почему GitHub Pages, а не VPS

- 0 ₽
- Деплой через git push
- HTTPS из коробки
- Нет состояния → нет проблем со стейтом

**Когда переходить на VPS:** если понадобится серверная логика (авторизация, персонализация, admin panel).

## Разделение verified / unverified

Ошибочный факт хуже отсутствующего. Любой сомнительный термин → `expert_review.json` со статусом `unverified`. В UI — явная плашка. Переводить в verified только после подтверждения источником.
