## Метод A — HTML-тег (правка `<head>`)
Google даёт строку вида:
`<meta name="google-site-verification" content="ТОКЕН">`
- Вставить её в `<head>` **`docs/index.html`** (для URL-prefix верификации достаточно корневой страницы; по желанию — продублировать на dictionary/history/origins для единообразия).
- Коммит + пуш в main (Pages обновится; бот не задет — путь `docs/`).

## Метод B — HTML-файл (проще для GitHub Pages)
Google даёт файл вида `googleXXXXXXXX.html`.
- Положить его **как есть** в `docs/` (корень Pages → будет доступен на `…/hockey-platform/googleXXXXXXXX.html`).
- Коммит + пуш. Файл не трогать/не переименовывать.
