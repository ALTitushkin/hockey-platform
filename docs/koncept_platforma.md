## 0. Зачем это вообще — ядро ценности

Продукт растёт из одной нити, которая проходит через всю твою жизнь: **делать сложное понятным.** Не «ещё одно приложение про хоккей» — таких хватает. А **передача понимания** там, где её сейчас нет.

Конкретная боль, с которой всё началось: русскоязычный человек (фанат, игрок, начинающий тренер) утыкается в англоязычный хоккейный материал или таблицу статистики — и не понимает. Готового, человеческого, быстрого моста англ→рус нет. Аналитик из СКА сделал себе сайтик-костыль — значит боль реальна, а решения нет.

**Ценность = момент «ага» в голове пользователя.** Всё остальное (платформа, подписки, комьюнити) — это масштабирование этого момента, а не замена ему. Если на каждом этапе теряется «ага» — этап сделан зря.

---

## 1. Принцип масштабируемости (читать до этапов)

Чтобы система росла, а не переписывалась каждый раз, закладываем три правила в фундамент:

**Правило 1. Одно ядро — много обёрток.**
«Словарь» — это не интерфейс, это база знаний (термин → категория → объяснение → сокращение → визуал → источник). Поверх неё можно надеть что угодно: поиск на сайте, Telegram-бота, карточки, медиа-контент. Меняешь обёртку — ядро не трогаешь. Это и есть масштабируемость на уровне архитектуры.

**Правило 2. Граница «AI / человек» проведена с первого дня.**
- **AI (Claude) уверенно делает:** проверяемые термины англ→рус, конспекты из открытых источников, структурирование, оформление.
- **Только живой эксперт:** русский сленг (рыбачок, поплавок), спорные трактовки, авторская методика.
- Это твой же принцип «агенту действия, человеку ответственность», применённый к продукту. Никогда не выдаём догадку за факт — на этом держится доверие, а доверие здесь единственный актив.

**Правило 3. Каждый этап самоценен и переживает паузу.**
Ты работаешь пару часов в неделю без дедлайнов. Поэтому каждый этап спроектирован так, что если следующего не будет — предыдущий всё равно остаётся рабочим и полезным. Никаких «это заработает только когда достроим всё».

## 2. Этапы: от зерна к системе

Пять этапов. Каждый — отдельная ступень с честной оценкой. Не нужно проходить все. Нужно проходить столько, сколько даёт энергию.

---

### Этап 1 — Ядро-словарь (СЕЙЧАС)
**Что:** база проверяемых терминов англ→рус (статистика + тактика), уже начата в HTML-черновике.

| | |
|---|---|
| **Насколько реально** | Уже сделано наполовину. Полностью реально. |
| **Чего стоит** | Несколько вечеров с Claude. 0 ₽. |
| **Где ресурсы** | Ты + Claude. Источники — открытые глоссарии (NHL, Natural Stat Trick, профильные разборы). |
| **Самоценность** | Даже если остановиться здесь — это рабочий справочник, которым ты сам пользуешься. |
| **Риск** | Минимальный. Единственное — не тащить в ядро непроверенный сленг. |

**Результат этапа:** аккуратная база на 30–50 терминов + список «на выверку с экспертом».

---

### Этап 2 — Удобный вход: поиск + бот
**Что:** превратить «листать энциклопедию» в «спросил — получил». Поле поиска на странице (печатаешь термин рус или англ → мгновенная карточка) и/или Telegram-бот с той же логикой.

> Это прямое следствие твоего замечания про интерфейс: энциклопедия — режим обучения, поиск/бот — режим инструмента. Нужны оба, но точка входа — быстрый ответ.

| | |
|---|---|
| **Насколько реально** | Реально. Mini App ты уже делал — стек знаком. |
| **Чего стоит** | Вечер–два с Claude Code на поиск; бот — отдельный вечер. 0 ₽ (хостинг бота — копейки или бесплатные тарифы). |
| **Где ресурсы** | Ты + Claude Code. |
| **Самоценность** | Бот — это уже продукт, которым могут пользоваться другие. Первый выход «к людям» (твой канал — готовая аудитория для теста). |
| **Риск** | Низкий. Технический, не смысловой. |

**Результат этапа:** живой инструмент, который не стыдно показать. Первая обратная связь от реальных людей.

---

### Этап 3 — Выверка и наполнение (вход эксперта)
**Что:** подключить живого носителя (кандидат №1 — аналитик из СКА), чтобы выверить сленговый слой и расширить базу. Здесь же — первое наполнение тактики визуалом (схемы, короткие видео).

| | |
|---|---|
| **Насколько реально** | Реально, но зависит от человека — это уже не только ты. |
| **Чего стоит** | Время на договорённость + твоё время на оформление. Деньги — по желанию (можно на бартер/соавторство). |
| **Где ресурсы** | Внешний эксперт. **Это первая точка, где нужен кто-то кроме тебя и Claude.** |
| **Самоценность** | База становится доверенной — теперь её можно показывать как авторитетный источник. |
| **Риск** | Зависимость от чужого согласия. Митигация: к этому этапу у тебя уже есть готовый продукт (этапы 1–2), с которым проще приходить к эксперту — «вот что я сделал, помоги выверить», а не «давай с нуля». |

**Результат этапа:** доверенная база + первые визуалы. Продукт с экспертной валидацией.

---

### Этап 4 — Образовательный слой
**Что:** не отдельные термины, а **структурированная база теории.** Находим открытые/разрешённые материалы (рус + зарубежные: канадские, американские, шведские, финские школы), перерабатываем через AI в оригинальные структурированные конспекты, ранжируем по уровню (новичок → продвинутый) и в перспективе по роли (игрок / тренер / менеджер). Тот же контент — в медиа-формате (карточки, шортсы).

> ⚠️ **Важная рамка — авторские права.** Нельзя взять чужую книгу/статью, перевести и опубликовать — это чужая интеллектуальная собственность. **Можно:** пересказ своими словами с указанием источника, оригинальные конспекты «по мотивам», работа с материалами, разрешёнными к использованию, собственная методика. Эту рамку закладываем в архитектуру здесь, а не чиним потом. При сомнении — консультация юриста до публикации.

| | |
|---|---|
| **Насколько реально** | Технически — да. Юридически — при соблюдении рамки выше. Это уже не «вечер», а регулярная работа. |
| **Чего стоит** | Существенное время. Возможно — первые небольшие расходы (дизайн, видео). Десятки тыс. ₽ при желании ускориться, но можно и в ноль на своём труде. |
| **Где ресурсы** | Ты + Claude + эксперт(ы) для проверки. Твои накопления позволяют вложиться, но **на этом этапе деньги не обязательны** — обязательно время и энергия. |
| **Самоценность** | Это уже образовательный ресурс, а не справочник. Можно собирать вокруг него аудиторию. |
| **Риск** | Средний: авторские права + объём работы. Митигация: начинать с узкой темы (например, «основы тактики для новичка»), не пытаться покрыть всё сразу. |

**Результат этапа:** структурированная теория по уровням. Основа для монетизации.

---

### Этап 5 — Платформа и монетизация (ГОРИЗОНТ)
**Что:** подписка (Boosty/аналог), комьюнити для обмена опытом, в перспективе — коллаборации с университетами / MBA Авангарда / хоккейными школами. Превращение фреймворка в самостоятельный продукт.

| | |
|---|---|
| **Насколько реально** | Реально как горизонт. Это **другой класс задач:** партнёрства, юрлицо, переговоры, возможно команда. |
| **Чего стоит** | Время + деньги + люди. Здесь впервые «вечер с Claude» не работает. |
| **Где ресурсы** | Партнёрства, возможно соосновател(и), возможно часть твоих накоплений как инвестиция. Комьюнити как источник и контента, и дохода. |
| **Самоценность** | Это и есть «операционная модель мечты»: продукт + процесс вокруг него + фреймворк, который сам становится продуктом. |
| **Риск** | Высокий, но к этому моменту он подкреплён всем, что построено на этапах 1–4. В это не прыгают — до этого дорастают. |

**Результат этапа:** самостоятельная платформа с доходом и сообществом.

## 3. Карта «реально / стоит / ресурсы» одним взглядом

| Этап | Реальность | Стоимость (деньги) | Стоимость (время) | Нужен кто-то кроме тебя? |
|------|-----------|-------------------|-------------------|--------------------------|
| 1 · Ядро-словарь | ✅ делается | 0 ₽ | вечера | Нет |
| 2 · Поиск + бот | ✅ знаком | ~0 ₽ | вечер-два | Нет |
| 3 · Выверка + визуал | ⚠️ зависит от эксперта | 0–бартер | время | **Да — эксперт** |
| 4 · Образ. слой | ⚠️ + авторские права | 0–десятки тыс. | регулярно | Эксперт для проверки |
| 5 · Платформа | 🔭 горизонт | значимо | много | **Да — партнёры/команда** |

**Граница «вечер с Claude» проходит между этапом 2 и 3.** До неё — ты сам. После — появляются люди и обязательства. Это естественная точка, где стоит остановиться и честно спросить себя, есть ли энергия идти дальше, или продукт хорош и таким.

## 4. Где какой инструмент Claude

- **Claude (чат):** думать, разбирать термины, писать объяснения, держать концепт. Этапы 1, 3, 4.
- **Claude Code:** поиск на странице, Telegram-бот, любая реализация. Этап 2, частично 4.
- **Claude Cowork:** когда наполнение станет повторяемым конвейером (нашёл материал → переработал → оформил → выложил) — собрать этот процесс как систему. Этап 4. Это и есть зачаток «виртуальной команды».

> Точные возможности и лимиты этих продуктов меняются — когда дойдёшь до конкретного инструмента, стоит свериться с актуальной документацией перед выбором.

---

## 5. Следующий шаг (один, не десять)

Не планировать все пять этапов. **Закрыть этап 1 до конца:** довести ядро-словарь до 30–50 проверенных терминов. Когда листаешь и чувствуешь «этим я бы сам пользовался» — этап закрыт, переходим к поиску/боту.

Если на каком-то шаге пропадёт энергия — это не провал. Это сигнал свериться: тот ли это «ага», ради которого всё затевалось. Продукт, остановленный на этапе 2, — полноценный продукт. Не каждый должен дорасти до этапа 5.

## 6. Продуктовая модель и IA — зафиксировано 14.06.2026 (v0.2)

> Уточнение перед редизайном (Sprint 5, Трек B). Не отменяет этапы выше — конкретизирует, КАК устроено ядро и куда оно растёт. Из этого раздела собирается ТЗ UX/UI-дизайнеру.

### 6.1 Формула платформы
**Вопрос → краткое выверенное саммари → якорь(и) в конкретное место каноничного материала.** Это выросший «ага-момент»: от «что такое форчек» до «как поставить зонную оборону» и «как стать скаутом» — один механизм, разная глубина ответа. Словарь, история, маршруты — не разные продукты, а разная глубина ответа на вопрос.

### 6.2 Без LLM (пока)
Движок — **структура + точные якоря**, не языковая модель. Надёжнее (нечему галлюцинировать), дешевле в поддержке, в духе принципа «не выдавать догадку за факт». LLM — на горизонт, если структура перерастёт ручную настройку. Следствие: бот/поиск отдаёт саммари + deep-link на якорь, а не «почитай главу целиком».

### 6.3 Три слоя амбиции (не смешивать)
- **Ядро (строим сейчас):** вопросы-ответы — термины, история, первые учебные маршруты.
- **Горизонт (проектируем как пустые слоты в IA, не наполняем):** роудмэпы тренера/скаута, материалы игрокам, дорожные карты «по мотивам» лучших практик.
- **Север (только направление, НЕ проектируем):** вузы / MBA-Авангард, агрегатор проблем хоккея и адвокаси. IA не должна мешать прикрутить это позже.

### 6.4 Информационная архитектура
Ответ-вход (поле «спроси») на главной + **равноправные** разделы: Словарь · История · (слот) Обучение · (слот) Сообщество. История перестаёт быть «подвалом».

### 6.5 Таксономия: две независимые оси
Текущие теги `новичок / болельщик / аналитик` смешивают уровень и роль («щелчок = аналитика» — симптом). Развести:
- **Уровень** (прогрессия «простое → сложное»): база → продвинутый.
- **Профиль** (кто / зачем пришёл): общий, игрок, тренер, аналитик, скаут…

Термин несёт обе метки одновременно. Нужен письменный **рубрикатор присвоения уровней** + ревизия текущих 49 терминов (контент/данные, не дизайн).

### 6.6 Связи и якоря в данных
Термины и контент получают: **вложенность/кластеры** (зонтик ⊂ большинство; кистевой/щелчок/подщёлкивание ⊂ броски), блок **«см. также»**, и **якоря-id** на конкретные места в текстах. Правка схемы `terms.json` + реестра истории (`history.json`).

### 6.7 Визуализация — первоклассный контент
Не украшение: у понятия может быть схема и/или видео прямо в ответе. Старт есть (`docs/diagrams/`). Реалистично near-term: **SVG-схемы делаем сами**, **видео встраиваем чужое** (YouTube + таймкод как ещё один якорь), оригинальный продакшн — на горизонт.

### 6.8 Связка с каналом
**Платформа — ядро; канал и медиа — инструменты дистрибуции**, не наоборот. Сейчас выглядит обратно — это переходное состояние.

### 6.9 Имя и бренд (решено 14.06)
Зонтичный бренд платформы — **«Культура хоккея»**, с позиционированием «при поддержке канала „Хоккейный Овертайм“». Канал не переименовываем — он медиа-рукав и воронка; платформа получает собственное имя-ядро. Бот — «Учебник хоккея».
