# Схема данных — hockey-platform

> Источник правды о структуре `terms.json` и `expert_review.json`.
> При изменении схемы — обновить `CLAUDE.md` тоже.

---

## terms.json

Массив объектов. Каждый объект — один хоккейный термин.

| Поле | Тип | Обязательное | Описание |
|------|-----|:---:|---------|
| `id` | string | ✓ | Уникальный slug (латиница, строчные, дефисы). Пример: `expected-goals` |
| `en` | string | ✓ | Каноническое английское название |
| `ru` | string | ✓ | Русский перевод или транслитерация |
| `category` | string | ✓ | `stat` / `tactic` / `position` / `rule` |
| `abbr` | string[] | ✓ | Массив сокращений. Пустой `[]` если нет |
| `definition` | string | ✓ | Объяснение по-человечески. Без академизма. 2–5 предложений |
| `ru_slang` | string\|null | ✓ | Разговорный русский аналог. `null` если нет |
| `visual` | string\|null | — | URL или описание визуала. `null` пока нет |
| `visual_type` | string\|null | — | `scheme` / `video` / `gif` / `null` |
| `source` | string | ✓ | Источник или сайт для проверки |
| `status` | string | ✓ | `verified` — проверено. `unverified` — требует проверки |
| `added` | string | ✓ | Дата добавления в формате ISO: `2026-06-07` |
| `notes` | string | — | Служебные заметки. Не отображаются в UI |

### Категории

- **stat** — статистические метрики (Corsi, xG, PDO, Sv%)
- **tactic** — тактические концепции (форчек, бэкчек, трап, слот)
- **position** — позиции и роли (winger, center, stay-at-home D)
- **rule** — правила игры (вбрасывание, офсайд, icing)

### Правила заполнения

1. `id` уникален в пределах файла
2. `definition` — без Wikiped-стиля. Пишем как объясняем другу
3. Если термин не уверен на 100% — в `expert_review.json`, не сюда
4. Аббревиатуры перечисляем все варианты: `["CF", "CA", "CF%"]`

## expert_review.json

Та же схема, что `terms.json`. Разница:
- `status` всегда `"unverified"`
- `notes` содержит причину неопределённости и что нужно уточнить
- В UI отображаются с плашкой «⚠ требует проверки»

### Как перевести термин из expert_review в terms

1. Получить подтверждение от эксперта (тренер, аналитик, профессиональный источник)
2. Заполнить поле `source` с указанием реального источника
3. Изменить `status` на `"verified"`
4. Переместить объект из `expert_review.json` в `terms.json`
5. Закоммитить: `data: verified термин fisherman (source: тренер Иванов)`
