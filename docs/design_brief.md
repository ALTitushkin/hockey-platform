## Промпт

Сделай редизайн сайта-словаря хоккейных терминов: https://altitushkin.github.io/hockey-platform/

**Продукт.** Англо-русский словарь хоккейных терминов (статистика, тактика, правила, позиции). Ядро сценария: человек встретил незнакомый термин в англоязычной статье → нашёл за 3 секунды → понял. Аудитория: русскоязычные хоккейные фанаты, игроки-любители, начинающие тренеры. Канал привлечения — Telegram-канал «Хоккейный Овертайм», так что сайт должен ощущаться его продолжением.

**Стиль.** За основу возьми приложенное лого канала: винтажная спортивная эмблема — вратарская маска, скрещённые клюшки, рукописная дуговая надпись. Хочу эстетику ретро-хоккейной программки / классических клубных афиш НХЛ 50–70-х: палитра от лого (стальной синий, графит, тёмно-синий) на тёплом кремовом фоне вместо стерильного белого, плотная акцидентная типографика для заголовков (кириллица обязательна), чернильные контуры и лёгкая печатная фактура — но дозированно, читабельность важнее декора. Что НЕ хочу: глянцевые градиенты, неоновое свечение, стеклянные карточки, стоковые AI-паттерны — текущий сайт именно этим и страдает.

**Структура — сохранить как есть, перерисовать подачу:**
- шапка с названием и коротким слоганом;
- поле поиска — главный элемент экрана, сразу под шапкой, крупное;
- фильтры-чипсы категорий: Статистика, Тактика, Правила, Позиции;
- сетка карточек терминов: английское название + русское, бейдж категории, сокращения (CF%, xG…), текст определения, иногда строка «сленг»;
- отдельная секция внизу «на выверке» — термины с плашкой «требует проверки экспертом», визуально приглушённые;
- футер со ссылкой на Telegram-канал и GitHub.

**Требования.** Mobile-first: основной трафик с телефона из Telegram. Карточек на странице 35+, поэтому сетка должна держать плотность без каши. Тёмная тема не нужна. Один экран = одна страница, без многостраничности.

**Формат результата:** статичная HTML-страница (один файл, CSS внутри), семантическая разметка. Карточки терминов и чипсы категорий сделай повторяющимися блоками с одинаковой структурой классов — содержимое потом будет подставляться скриптом из JSON, поэтому достаточно 4–6 карточек-примеров с реалистичным содержимым (Corsi, Expected Goals, Форчек, Слот, Тафгай — текст возьми с текущего сайта).

## Нюансы работы с Claude Design (первый раз)

1. **Прикрепи лого файлом** к первому сообщению — текстовое описание стиля работает хуже референса.
2. **Используй web capture** на текущий сайт (если предложит) — так Claude Design увидит реальную структуру, а не выдумает свою.
3. **Не проси всё сразу идеально.** Первая выдача — черновик. Дальше: инлайн-комментарии на конкретных элементах («этот заголовок легче», «фон теплее») работают лучше, чем общие «сделай красивее».
4. **Попроси 2–3 направления** в начале («покажи 3 варианта настроения: строгий ретро, тёплый ламповый, современный с ретро-акцентами») — выбрать проще, чем формулировать вслепую.
5. **Ползунки (adjustment knobs)** — проси их для спорных параметров: насыщенность фактуры, плотность сетки, размер шрифта.
6. **Экспорт — standalone HTML.** Не Canva, не PDF. Этот файл потом отдай Claude в Cowork: интеграция с поиском (Fuse.js) и базой terms.json — его часть работы.
7. **Проверь мобильную ширину** перед экспортом — сузь превью до ~390px.
